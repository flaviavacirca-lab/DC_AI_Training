Copilot 101: The Basics

Everything you need to know to start using Microsoft Copilot effectively — where to find it, how it thinks, and how to get the most from every conversation.

What you'll learn

  • Where to find Copilot and how to access it at Denneen
  • The Web vs. Work toggle and when to use each mode
  • How chat memory works and when to start a new conversation
  • Common first-timer mistakes and how to avoid them

What Is Microsoft Copilot?

Microsoft Copilot is an AI assistant built into your Microsoft 365 tools. Think of it as a very capable colleague who can draft content, summarize documents, analyze data, and brainstorm ideas — but only when you give it clear direction.

At Denneen, you access Copilot through the M365 Copilot app on your laptop. Inside that app, there's a Web/Work toggle that controls what Copilot can reference. Understanding that toggle is the first step to getting real value from it.

Your First 5 Minutes with Copilot

Not sure where to start? Try these tasks right now to get comfortable.

1

Open the M365 Copilot app

Find the Copilot app on your laptop (it's in your Start menu or taskbar). Sign in with your work account if prompted. You'll see the chat interface with the Web/Work toggle at the top.

2

Try a "Web" mode query

Make sure the toggle is set to Web. Try asking: "What are the top 3 trends in management consulting in 2026?" Notice how Copilot pulls from public sources.

3

Switch to "Work" mode

Flip the toggle to Work. Try asking: "Summarize the most recent emails from [a colleague's name]." Notice how Copilot now pulls from your M365 data — emails, files, and meetings.

4

Iterate on a response

Stay in the same chat and ask Copilot to adjust its response: "Make that shorter" or "Now rewrite it as bullet points for a slide deck." This is where the real power is — building on each exchange.

The Web / Work Toggle: What It Actually Does

Inside the Microsoft 365 Copilot app, you will see a toggle at the top of the chat interface labeled Work and Web. This toggle changes what Copilot primarily uses to answer your question. Think of it as choosing the main source of information Copilot should rely on.

💻

Where You Are

Open the M365 Copilot app on your laptop. At the top of the chat window you will see a toggle labeled: Work | Web. This is the standard interface used across the firm. You are not opening a browser — this is Copilot inside the Microsoft 365 environment.

Work Mode

Primary source: Your Microsoft 365 data.

Copilot can access information you already have permission to see, including:

  • SharePoint documents
  • OneDrive files
  • Outlook emails
  • Teams chats
  • Calendar events
  • Meeting transcripts (when available)

This works through the Microsoft Graph, which allows Copilot to reason over your work environment.

Web Grounding in Work Mode

When you use Work mode, Copilot's primary grounding source is always your Microsoft 365 data. However, Copilot can optionally supplement its answers with public web information — this is called web grounding.

How it works: If the web search plugin is enabled (controlled by your tenant admin or your personal Copilot settings), Copilot may pull in publicly available information from the Bing index alongside your work data. This happens automatically when Copilot determines that web context would improve the response.

What this means in practice:

  • You ask Copilot to draft a competitive analysis using an internal SharePoint document — it may also pull in recent public news about the competitor to enrich the output
  • You ask Copilot to summarize a client email thread — it stays entirely within your M365 data because no web context is needed
  • You ask Copilot about an industry trend referencing your internal notes — it may blend your notes with current public data

Key point: Your work data always remains the primary context. Web grounding only adds supplementary public information — it never replaces or deprioritizes your organizational data. If web search is disabled by your admin, Work mode uses M365 data exclusively.

Best for:

  • Working with internal files
  • Drafting emails using existing documents
  • Summarizing meetings or message threads
  • Finding information across your work environment

Think of it as...

A colleague who can instantly read through your emails, files, meetings, and chats and help you work with them.

Web Mode

Primary source: Public information from the internet.

In this mode, Copilot generates answers using web content indexed by Bing and general knowledge. It does not access your Microsoft 365 files, emails, or internal company content.

Best for:

  • Industry research
  • Learning new frameworks or concepts
  • Market exploration
  • Public company comparisons
  • Brainstorming ideas

Reminder

When using Web mode, follow the Denneen AI Policy for what's appropriate to share. Do not paste confidential client data into a Web-mode prompt.

Quick Comparison

Mode Primary Information Source Can Access Your Work Files
Work M365 data (SharePoint, Outlook, Teams, OneDrive) Yes
Web Public internet content No

Work mode may also include web grounding if web search is enabled, but it always prioritizes your work context.

Work Mode Examples

  • "Draft an email based on the strategy deck in SharePoint." → Copilot pulls the deck from your SharePoint and drafts the email.
  • "Summarize the last five emails from [client name]." → Copilot reads your Outlook and creates the summary.
  • "What action items came out of Tuesday's project meeting?" → Copilot checks the meeting transcript and Teams chat.

Web Mode Examples

  • "Summarize recent public trends in the pet food industry." → Copilot searches public sources and gives you an overview.
  • "What is the MECE framework and how is it used in consulting?" → Copilot explains based on general knowledge.
  • "Compare Chewy and Petco using recent public financial data." → Copilot pulls from publicly available data.

Rule of Thumb

  • Working with internal files or client materials stored in M365? → Use Work
  • Doing market research or learning about a new concept? → Use Web
  • Not sure? → Start in Work mode and follow the Denneen AI policy for data sharing.

How Chat Memory Works

Understanding what Copilot remembers — and what it doesn't — is key to using it well.

1

Within a single chat: Copilot remembers everything

As long as you stay in the same conversation, Copilot can see everything you've said and everything it has responded with. It builds context as you go.

Example: If you say "Summarize this report" and then later say "Now turn that summary into an email to the client," Copilot knows what "that summary" refers to.
2

Between chats: Copilot starts fresh

When you open a new conversation, Copilot has no memory of previous chats. It doesn't carry over context, files, or instructions from earlier sessions.

Example: If you spent 20 minutes refining a competitive analysis in one chat, then start a new chat asking "Can you update that analysis?" — Copilot won't know what you're referring to.
3

Context window has limits

Even within a single chat, there's a limit to how much Copilot can hold in memory. Very long conversations may cause it to "forget" earlier details. If your responses start losing quality, it may be time to start a new chat with a fresh summary.

Tip: If a conversation is getting long, copy the key outputs and start a new chat. Paste in a summary like: "Here's what we've established so far: [paste key points]. Now let's continue with..."

Same Chat or New Chat?

This is one of the most practical decisions you'll make. Here's a simple guide.

Stay in the Same Chat When...

  • You're iterating on the same output (e.g., refining an email draft)
  • Your next request builds directly on the previous response
  • You need Copilot to remember specific instructions you gave earlier
  • You're in the middle of a multi-step workflow (e.g., research → analysis → deliverable)
  • You're referencing files or data you've already shared in this session

Start a New Chat When...

  • You're switching to a completely different topic or project
  • The current conversation is getting long and Copilot's responses are declining in quality
  • You want a "fresh perspective" without prior assumptions
  • The chat has accumulated instructions that no longer apply (e.g., you told it to write in a formal tone but now need casual)
  • You're done with one task and starting something unrelated

Pro Tip: The "Carry-Over Summary"

When you need to start a new chat but want to preserve context, use this approach:

Here's the context from our previous work:
- Project: [project name]
- Client: [client details]
- What we've done so far: [key decisions and outputs]
- What we're working on now: [current task]

Please continue from here and [next action].

Where to Find Copilot in Your Apps

A quick overview of where Copilot lives in each M365 app and what it does best there.

📧

Outlook

Summarize long email threads, draft replies, and adjust tone. Look for the Copilot icon when composing or reading an email.

Try: "Summarize this thread and highlight any action items for me."

📝

Word

Draft new documents, rewrite sections, summarize long documents, or transform content. Access Copilot from the ribbon or the side panel.

Try: "Draft an executive summary of this document in 200 words."

📊

Excel

Analyze data, create formulas, build charts, and spot trends. Copilot works best with data formatted as tables.

Try: "What are the top 5 trends in this data? Add a chart."

📊

PowerPoint

Create presentations from scratch, from a Word doc, or from an outline. Also useful for reorganizing and summarizing existing decks.

Try: "Create a presentation from [document name] with 10 slides."

💬

Teams

Get meeting summaries, catch up on chats, and find information across channels. Copilot can recap meetings you missed and extract action items.

Try: "What decisions were made in today's meeting and who has action items?"

🔍

Microsoft 365 Chat

The cross-app assistant that can search across your emails, files, meetings, and chats simultaneously. Access it from the M365 app or Teams.

Try: "Find the latest version of the strategy deck for [client] and summarize the key recommendations."

Common First-Timer Mistakes

✗ Using Web mode for tasks that need your files
✓ Switch the toggle to Work when you need internal data

Web mode can't see your SharePoint, emails, or Teams. If your task involves firm or client materials, flip the toggle to Work.

✗ Starting a new chat for every follow-up question
✓ Stay in the same chat when building on previous outputs

Every new chat wipes Copilot's memory. If you're refining something, keep going in the same conversation.

✗ Asking one-word questions like "Summarize"
✓ Tell Copilot what to focus on, who the audience is, and how to format the output

Vague prompts get vague responses. Even a few extra words of context dramatically improve the result.

✗ Trusting everything Copilot says without checking
✓ Verify data points, client names, and any specific claims

Copilot is confident even when it's wrong. Always review outputs before sharing them with clients or colleagues.

Practice: Try It Yourself

Spend 5 minutes on one of these exercises to solidify what you just learned.

Exercise 1: Web vs. Work

Open Copilot and try the same question in both modes:

What are the top trends in the consulting industry for 2025-2026?

First in Web mode, then in Work mode. Notice how the answers differ — Web pulls from the internet, Work pulls from your M365 files.

Exercise 2: Refine in the Same Chat

Ask Copilot a broad question, then refine it without starting a new chat:

Summarize the key challenges facing mid-market B2B companies.

Then follow up: “Focus specifically on customer retention challenges. Format as 5 bullet points I could use in a presentation.” See how the context carries over.

Exercise 3: Vague vs. Specific

Try a vague prompt, then a specific one for the same task:

Vague: “Help me with an email.”

Specific: “Draft a 3-sentence email to a VP-level client confirming our next steering committee meeting on Friday at 2pm. Professional tone, include an agenda placeholder.”

Compare the results. This is the single biggest lever for better Copilot output.
