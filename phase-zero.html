Phase Zero Training

Management Interviews & Data Review — Copilot-Enabled Standard

What this training covers

This training defines the required, standard way to use Microsoft Copilot during Phase 0 for:

  1. Management Interviews
  2. Data Review

Objective

  • Reduce manual effort
  • Improve consistency and defensibility
  • Avoid missing information
  • Accelerate Phase 0 synthesis and scoping

This is a process change, not optional “AI experimentation.”

Why Phase Zero Breaks Without This Process

The Problem

Phase 0 work often fails because:

  • Insights get scattered across notebooks, transcripts, and individual notes with no consistent structure
  • Synthesis takes too long because every consultant formats and organizes their notes differently
  • Phase 1 scope decisions end up relying on incomplete information because there is no systematic way to verify coverage

How This Process Solves It

This Copilot-enabled process addresses each issue directly:

  • Structured interview capture — every interview produces the same outputs in the same format, stored in the same location
  • Rapid synthesis across interviews — standardized theme tables allow cross-interview analysis in minutes, not days
  • Evidence-backed foundation for Phase 1 — defensible data inventory and gap analysis ensure scope decisions are grounded in what the data actually shows

Section 1

Phase Zero — Management Interviews

Purpose

Management interviews are a core input into Phase 0 findings and Phase 1 scope decisions. This process ensures interviews are:

  • Captured consistently
  • Summarized quickly
  • Defensible with evidence
  • Easy to synthesize across stakeholders

Tools Used (Copilot-Only)

  • Microsoft Teams (recording + transcription)
  • SharePoint
  • Copilot Chat
  • (Optional, leads only) Copilot Notebook

Role Ownership

Each step in the interview process has a clear owner.

Step Owner
Interview guide upload Workstream Lead
Recording Interviewer
Summary generation Interview notetaker
Theme tagging Analyst / Consultant
Theme synthesis Consultant / Workstream Lead

Required Folder Structure

Create this structure before interviews begin.

/Phase_0 /01_Interview_Materials /02_Interview_Recordings /03_Interview_Transcripts /04_Interview_Summaries /05_Interview_Theme_Log

Copilot will only work well if files are stored consistently. Create this folder structure once and use it for every engagement.

Step-by-Step Process

Step 1: Upload the Approved Interview Guide ~5 min

Who: Project Lead File upload: Yes

Upload the final interview guide to:

/01_Interview_Materials/Approved_Interview_Guide.docx

Why this matters

  • Copilot needs to understand what questions were asked
  • This prevents generic or off-target summaries

Step 2: Record and Transcribe Every Interview ~5 min post-interview

Who: Interviewer File upload: Yes (manual)

At the start of every Teams interview:

  • Turn Recording ON
  • Turn Transcription ON

After the meeting:

  1. Locate the recording and transcript
  2. Upload them manually to:
/02_Interview_Recordings /03_Interview_Transcripts

Important: Teams does not reliably auto-save to your project folder. Manual upload is required.

Step 3 (Optional): Create a Copilot Notebook — Leads Only ~5 min

Who: Phase 0 Lead File upload: Yes (add sources)

Notebook name:

Phase_0_Management_Interviews_[ClientName]

Add only:

  • Approved interview guide
  • All interview transcripts

Why this helps

  • Keeps Copilot focused only on interviews
  • Improves cross-interview synthesis
  • Reduces risk of unrelated context

If you are summarizing individual interviews only, you may skip this step.

Step 4: Generate the Interview Summary (Required) ~3–5 min per interview

Who: Interview owner Where: Copilot Chat (or Notebook if used) File upload: Yes (explicitly reference files)

Prompt (copy exactly):

Use ONLY the following documents: - Approved_Interview_Guide.docx - [IntervieweeName]_Transcript.docx Do not use emails, chats, or other files. Create a 1-page Phase 0 interview summary with: 1. Top 5 takeaways 2. Key quotes with timestamps 3. Explicit knowledge or data gaps mentioned 4. Documents or analyses referenced by the interviewee 5. Any tensions or contradictions with prior interviews (if evident) Use evidence only and cite the document or transcript location for each finding. Do not infer. Use only what was stated.

Saving the output

Copilot Chat does not generate downloadable Word files. You must:

  1. Copy the output
  2. Paste into a clean Word document
  3. Save as: /04_Interview_Summaries/[IntervieweeName]_[Role]_Summary.docx

Human Review Required

Before using this summary, verify:

  • Quotes appear in the transcript
  • Timestamps are correct
  • The summary reflects the interview tone
  • No conclusions were inferred beyond what was said

Step 5: Tag Interview Themes (Standardized Table) ~3–5 min per interview

Where: Copilot Chat File upload: Yes (summary + transcript)

This step ensures all interviews produce identical, copy-paste-ready output.

Prompt (structure must not change):

Using ONLY: - [IntervieweeName]_Summary.docx - [IntervieweeName]_Transcript.docx Return the output ONLY as a table with the following columns: 1. Interviewee Name 2. Interviewee Role 3. Takeaway ID (T1, T2, T3…) 4. Takeaway Summary 5. Theme Category (Market, Customer, Competitive, Business Performance, Strategy / Decision-Making, Risks / Watchouts) 6. Supporting Quote (verbatim) 7. Timestamp (MM:SS) Rules: - Do not add, remove, rename, or reorder columns - Do not include commentary outside the table - Every takeaway must include a quote and timestamp - Use evidence only and cite the transcript location for each finding

Saving instructions

  1. Copy the table
  2. Paste into Excel
  3. Save to: /05_Interview_Theme_Log/Interview_Theme_Log.xlsx

Human Review Required

Before using this table, verify:

  • Themes match the actual interview content
  • Quotes support the assigned themes
  • Theme categories are accurate
  • Timestamps correspond to the correct quotes

Step 6: Cross-Interview Theme Synthesis ~10–15 min

Who: Consultant / Workstream Lead Where: Copilot Chat or Notebook File upload: Yes (all summaries + theme log)

This step identifies recurring patterns, points of agreement, and contradictions across all completed interviews. It is the bridge between individual interview outputs and Phase 1 hypothesis development.

Prompt:

Using ONLY the following files: - All interview summaries in /04_Interview_Summaries - Interview_Theme_Log.xlsx Identify: 1. Recurring themes across interviews (themes raised by 2+ interviewees) 2. Areas of agreement across stakeholders 3. Areas of contradiction or tension between stakeholders 4. Strategic risks mentioned by multiple interviewees 5. Topics requiring validation in Phase 1 Return a structured list of themes. For each theme, include: - Theme name - Which interviewees raised it - Supporting evidence (quotes or references) Use evidence only and cite the document or transcript location for each finding. Do not infer beyond what was stated in the interviews.

Saving instructions

  1. Copy the output
  2. Paste into a Word document
  3. Save as: /05_Interview_Theme_Log/Cross_Interview_Synthesis.docx

Human Review Required

Before using this synthesis, verify:

  • Themes genuinely appear across multiple interviews
  • Contradictions are real, not artifacts of summarization
  • Stakeholder positions are accurately represented
  • No conclusions were inferred beyond what interviewees stated

Management Interview Compliance Checklist

  • Interview guide uploaded
  • Recording ON
  • Transcript uploaded
  • Copilot summary created
  • Summary saved as Word doc
  • Theme table completed
  • Cross-interview synthesis completed

If any box is unchecked, the interview process is not complete.

Section 2

Phase Zero — Data Review

Purpose

Phase 0 data review determines:

  • What information exists today
  • What gaps remain
  • What work is required in Phase 1

This process ensures no documents are missed and all conclusions are defensible.

Tools Used (Copilot-Only)

  • SharePoint
  • Copilot Chat
  • Copilot Notebook (always recommended)

Required Folder Structure

/Phase_0 /06_Data_Request /07_Data_Received /08_Data_Inventory /09_Data_Summaries /10_RYG_Assessment

Step-by-Step Process

Step 1: Finalize the Phase 0 Data Request List ~10 min

Who: Project Lead File upload: Yes

Upload to:

/06_Data_Request/Phase_0_Data_Request_List.xlsx

This list defines what Phase 0 is assessing.

Step 2: Upload ALL Received Data ~5–15 min

Who: Project team File upload: Yes

Upload everything the client sends to:

/07_Data_Received

Do not pre-filter. Upload all received documents regardless of perceived relevance.

Step 3: Batch Document Analysis (Required for 20+ Documents) ~10–20 min

Method: Copilot Notebook

Using a Notebook avoids running prompts 80+ times individually.

Step 3A: Create the Data Review Notebook

Notebook name:

Phase_0_Data_Review_[ClientName]

Add only these sources:

  • Phase_0_Data_Request_List.xlsx
  • Entire folder /07_Data_Received

Step 3B: Master Document Analysis Prompt

You are supporting Phase 0 strategy readiness. Using ONLY: - Phase_0_Data_Request_List.xlsx - All documents in /07_Data_Received For EACH document: 1. Identify which specific data request items it supports 2. Explain why it supports those items 3. Identify which data request items it does NOT support 4. Extract and list ALL quantitative data included (numbers, KPIs, market sizes, growth rates, financials, dates) 5. Note the time period covered 6. Assign initial confidence (High / Medium / Low) Use evidence only and cite the document location for each finding. Outputs required: A. A clearly separated summary section for each document B. A consolidated table suitable for Excel export

Step 3C: Save Outputs

Copilot will not auto-generate Word files. You must:

  1. Copy all document summaries
  2. Paste into one master Word document
  3. Save as: /09_Data_Summaries/Phase_0_Master_Data_Summary.docx

Human Review Required

Before using document summaries, verify:

  • Numbers appear in the source document
  • Date ranges are interpreted correctly
  • Documents were mapped to the correct data request items
  • No unsupported conclusions were added

Step 4: Build the Data Inventory Table ~5 min

Where: Copilot Chat or Notebook File upload: Yes (master summary)
Using Phase_0_Master_Data_Summary.docx: Create a table with columns: 1. Document Name 2. Data Request Items Covered 3. Quantitative Data Present (Yes/No + examples) 4. Time Period Covered 5. Confidence Level Use evidence only and cite the document location for each entry. Return ONLY the table.

Save instructions

  1. Copy table
  2. Paste into Excel
  3. Save as: /08_Data_Inventory/Data_Inventory.xlsx

Human Review Required

Before using this inventory, verify:

  • Numbers appear in the source documents
  • Date ranges are interpreted correctly
  • Documents were mapped to the correct data request items

Step 5: Red / Yellow / Green Assessment ~5–10 min

Where: Copilot Chat or Notebook File upload: Yes (Data Request List + Data Inventory)

Excel is for formatting only — run the assessment in Copilot first.

Using ONLY: - Phase_0_Data_Request_List.xlsx - Data_Inventory.xlsx For each data request item: 1. Assign Red, Yellow, or Green 2. List documents reviewed 3. Explain the rating 4. Identify what work would close gaps Use evidence only and cite the document location for each rating. Do not assume missing data. Do not infer beyond evidence.

Save as

/10_RYG_Assessment/RYG_Assessment.xlsx

Human Review Required

AI can propose R/Y/G ratings, but the workstream lead must confirm:

  • Evidence supports each rating
  • Cited documents support the assessment
  • Missing data is correctly identified

Step 6: Phase 1 Data Gap Identification ~5–10 min

Where: Copilot Chat or Notebook File upload: Yes (Data Request List + Data Inventory + RYG Assessment)

This step identifies missing information that must be addressed in Phase 1. It translates R/Y/G ratings into actionable workstream recommendations.

Prompt:

Using ONLY: - Phase_0_Data_Request_List.xlsx - Data_Inventory.xlsx - RYG_Assessment.xlsx Identify: 1. All remaining Red data request items — what is completely missing and why 2. All remaining Yellow data request items — what is partially covered and what is still needed 3. For each Red and Yellow item, the specific analysis or data collection required to resolve it 4. Recommended Phase 1 workstreams based on the gaps identified Use evidence only and cite the document location for each finding. Do not assume data exists if it was not found in the inventory. Return as a structured table.

Save as

/10_RYG_Assessment/Phase_1_Data_Gaps.xlsx

Human Review Required

Before using gap analysis, verify:

  • Red/Yellow classifications are accurate
  • Missing data is genuinely missing, not misfiled
  • Recommended workstreams are actionable and realistic

Data Review Compliance Checklist

  • Data request list uploaded
  • All files uploaded
  • Notebook created
  • Master summary created
  • Data inventory exported to Excel
  • R/Y/G assessment completed
  • Phase 1 data gaps identified

If any box is unchecked, the data review is not complete.

AI Output Verification Checklist

Use this checklist before incorporating any Copilot output into your Phase 0 work.

Before using AI outputs

  • Quotes appear in the source document
  • Numbers match the original data
  • No unsupported conclusions were added
  • Sources are real and accessible
  • Output reflects the correct context

Phase Zero Is Complete When

Use this checklist to confirm all Phase 0 deliverables are finalized before proceeding to Phase 1.

  • All interviews summarized
  • Interview theme log completed
  • Cross-interview synthesis completed
  • All documents reviewed
  • Data inventory created
  • R/Y/G assessment finalized
  • Data gaps identified

If Copilot Doesn't Give You the Output You Expected

This happens. Do not abandon the process. Use these fixes.

Problem: Output is inconsistent

Add to your prompt: "Return ONLY the table. No commentary."

Problem: Documents were missed

Confirm the entire folder was added to the Notebook. Re-run with: "Confirm all documents were reviewed."

Problem: Output feels shallow

Add to your prompt: "Be exhaustive. This is for client-facing Phase 0 conclusions."

Problem: This feels like more work

Manual Phase 0 means repeated reading and rewriting across dozens of documents. With Notebook + one prompt, you get a single pass — and every downstream step (inventory, R/Y/G, synthesis) is faster because the foundation is already structured.
