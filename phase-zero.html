Phase Zero Training

Management Interviews & Data Review — Copilot-Enabled Standard

What this training covers

This training defines the required, standard way to use Microsoft Copilot during Phase 0 for:

  1. Management Interviews
  2. Data Review

Objective

  • Reduce manual effort
  • Improve consistency and defensibility
  • Avoid missing information
  • Accelerate Phase 0 synthesis and scoping

This is a process change, not optional "AI experimentation."

Section 1

Phase Zero — Management Interviews

Purpose

Management interviews are a core input into Phase 0 findings and Phase 1 scope decisions. This process ensures interviews are:

  • Captured consistently
  • Summarized quickly
  • Defensible with evidence
  • Easy to synthesize across stakeholders

Tools Used (Copilot-Only)

  • Microsoft Teams (recording + transcription)
  • SharePoint
  • Copilot Chat
  • (Optional, leads only) Copilot Notebook

Required Folder Structure

Create this structure before interviews begin.

/Phase_0 /01_Interview_Materials /02_Interview_Recordings /03_Interview_Transcripts /04_Interview_Summaries /05_Interview_Theme_Log

Copilot will only work well if files are stored consistently. Create this folder structure once and use it for every engagement.

Step-by-Step Process

Step 1: Upload the Approved Interview Guide

Who: Project Lead File upload: Yes

Upload the final interview guide to:

/01_Interview_Materials/Approved_Interview_Guide.docx

Why this matters

  • Copilot needs to understand what questions were asked
  • This prevents generic or off-target summaries

Step 2: Record and Transcribe Every Interview

Who: Interviewer File upload: Yes (manual)

At the start of every Teams interview:

  • Turn Recording ON
  • Turn Transcription ON

After the meeting:

  1. Locate the recording and transcript
  2. Upload them manually to:
/02_Interview_Recordings /03_Interview_Transcripts

Important: Teams does not reliably auto-save to your project folder. Manual upload is required.

Step 3 (Optional): Create a Copilot Notebook — Leads Only

Who: Phase 0 Lead File upload: Yes (add sources)

Notebook name:

Phase_0_Management_Interviews_[ClientName]

Add only:

  • Approved interview guide
  • All interview transcripts

Why this helps

  • Keeps Copilot focused only on interviews
  • Improves cross-interview synthesis
  • Reduces risk of unrelated context

If you are summarizing individual interviews only, you may skip this step.

Step 4: Generate the Interview Summary (Required)

Who: Interview owner Where: Copilot Chat (or Notebook if used) File upload: Yes (explicitly reference files)

Prompt (copy exactly):

Use ONLY the following documents: - Approved_Interview_Guide.docx - [IntervieweeName]_Transcript.docx Do not use emails, chats, or other files. Create a 1-page Phase 0 interview summary with: 1. Top 5 takeaways 2. Key quotes with timestamps 3. Explicit knowledge or data gaps mentioned 4. Documents or analyses referenced by the interviewee 5. Any tensions or contradictions with prior interviews (if evident) Do not infer. Use only what was stated.

Saving the output

Copilot Chat does not generate downloadable Word files. You must:

  1. Copy the output
  2. Paste into a clean Word document
  3. Save as: /04_Interview_Summaries/[IntervieweeName]_[Role]_Summary.docx

Step 5: Tag Interview Themes (Standardized Table)

Where: Copilot Chat File upload: Yes (summary + transcript)

This step ensures all interviews produce identical, copy-paste-ready output.

Prompt (structure must not change):

Using ONLY: - [IntervieweeName]_Summary.docx - [IntervieweeName]_Transcript.docx Return the output ONLY as a table with the following columns: 1. Interviewee Name 2. Interviewee Role 3. Takeaway ID (T1, T2, T3…) 4. Takeaway Summary 5. Theme Category (Market, Customer, Competitive, Business Performance, Strategy / Decision-Making, Risks / Watchouts) 6. Supporting Quote (verbatim) 7. Timestamp (MM:SS) Rules: - Do not add, remove, rename, or reorder columns - Do not include commentary outside the table - Every takeaway must include a quote and timestamp

Saving instructions

  1. Copy the table
  2. Paste into Excel
  3. Save to: /05_Interview_Theme_Log/Interview_Theme_Log.xlsx

Management Interview Compliance Checklist

  • Interview guide uploaded
  • Recording ON
  • Transcript uploaded
  • Copilot summary created
  • Summary saved as Word doc
  • Theme table completed

If any box is unchecked, the interview is not complete.

Section 2

Phase Zero — Data Review

Purpose

Phase 0 data review determines:

  • What information exists today
  • What gaps remain
  • What work is required in Phase 1

This process ensures no documents are missed and all conclusions are defensible.

Tools Used (Copilot-Only)

  • SharePoint
  • Copilot Chat
  • Copilot Notebook (required for scale)

Required Folder Structure

/Phase_0 /06_Data_Request /07_Data_Received /08_Data_Inventory /09_Data_Summaries /10_RYG_Assessment

Step-by-Step Process

Step 1: Finalize the Phase 0 Data Request List

Who: Project Lead File upload: Yes

Upload to:

/06_Data_Request/Phase_0_Data_Request_List.xlsx

This list defines what Phase 0 is assessing.

Step 2: Upload ALL Received Data

Who: Project team File upload: Yes

Upload everything the client sends to:

/07_Data_Received

Do not pre-filter. Upload all received documents regardless of perceived relevance.

Step 3: Batch Document Analysis (Required for 20+ Documents)

Method: Copilot Notebook

Using a Notebook avoids running prompts 80+ times individually.

Step 3A: Create the Data Review Notebook

Notebook name:

Phase_0_Data_Review_[ClientName]

Add only these sources:

  • Phase_0_Data_Request_List.xlsx
  • Entire folder /07_Data_Received

Step 3B: Master Document Analysis Prompt

You are supporting Phase 0 strategy readiness. Using ONLY: - Phase_0_Data_Request_List.xlsx - All documents in /07_Data_Received For EACH document: 1. Identify which specific data request items it supports 2. Explain why it supports those items 3. Identify which data request items it does NOT support 4. Extract and list ALL quantitative data included (numbers, KPIs, market sizes, growth rates, financials, dates) 5. Note the time period covered 6. Assign initial confidence (High / Medium / Low) Outputs required: A. A clearly separated summary section for each document B. A consolidated table suitable for Excel export

Step 3C: Save Outputs

Copilot will not auto-generate Word files. You must:

  1. Copy all document summaries
  2. Paste into one master Word document
  3. Save as: /09_Data_Summaries/Phase_0_Master_Data_Summary.docx

Step 4: Build the Data Inventory Table

Where: Copilot Chat or Notebook File upload: Yes (master summary)
Using Phase_0_Master_Data_Summary.docx: Create a table with columns: 1. Document Name 2. Data Request Items Covered 3. Quantitative Data Present (Yes/No + examples) 4. Time Period Covered 5. Confidence Level Return ONLY the table.

Save instructions

  1. Copy table
  2. Paste into Excel
  3. Save as: /08_Data_Inventory/Data_Inventory.xlsx

Step 5: Red / Yellow / Green Assessment

Where: Copilot Chat or Notebook File upload: Yes (Data Request List + Data Inventory)

Excel is for formatting only — run the assessment in Copilot first.

Using ONLY: - Phase_0_Data_Request_List.xlsx - Data_Inventory.xlsx For each data request item: 1. Assign Red, Yellow, or Green 2. List documents reviewed 3. Explain the rating 4. Identify what work would close gaps Do not assume missing data. Do not infer beyond evidence.

Save as

/10_RYG_Assessment/RYG_Assessment.xlsx

Data Review Compliance Checklist

  • Data request list uploaded
  • All files uploaded
  • Notebook created
  • Master summary created
  • Data inventory exported to Excel
  • R/Y/G assessment completed

If any box is unchecked, the data review is not complete.

If Copilot Doesn't Give You the Output You Expected

This happens. Do not abandon the process. Use these fixes.

Problem: Output is inconsistent

Add to your prompt: "Return ONLY the table. No commentary."

Problem: Documents were missed

Confirm the entire folder was added to the Notebook. Re-run with: "Confirm all documents were reviewed."

Problem: Output feels shallow

Add to your prompt: "Be exhaustive. This is for client-facing Phase 0 conclusions."

Problem: This feels like more work

Manual Phase 0 means repeated reading and rewriting across dozens of documents. With Notebook + one prompt, you get a single pass — and every downstream step (inventory, R/Y/G, synthesis) is faster because the foundation is already structured.
